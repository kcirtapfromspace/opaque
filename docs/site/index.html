Opaque

Approval-gated secrets broker for AI coding tools.

LLMs get operations (e.g., "set this GitHub secret"), never plaintext values. Every operation passes through a five-stage enforcement pipeline.

Policy→ Approval→ Execute→ Sanitize→ Audit
Get Started

Features

Deny-by-Default Policy

Allowlist-based rules with glob patterns on operations, client identity, target fields, and secret names. No rule means no access.

Biometric Approvals

Operation-bound native OS prompts: macOS Touch ID via LocalAuthentication and Linux polkit. Approval leases avoid prompt fatigue.

Typestate Sanitization

Compile-time enforcement that only sanitized responses leave the enclave. Secret-pattern scrubbing as defense-in-depth.

Structured Audit Log

Every request, approval, and outcome is recorded to SQLite with correlation IDs. Query with opaque audit tail.

MCP Server

First-class Claude Code integration via the Model Context Protocol. The opaque-mcp binary exposes only Safe operations over stdio.

Multiple Providers

GitHub Actions/Codespaces/Dependabot/org secrets, GitLab CI variables, 1Password, Bitwarden Secrets Manager, and HashiCorp Vault.

Client Identity

Identifies callers via Unix peer credentials, executable path/hash, and optional macOS code-signing Team ID.

Config Seal

SHA-256 integrity seal stored in the OS keychain. The daemon refuses to start if the policy has been tampered with.

Agent Session Wrapping

Run AI agents in scoped sessions with opaque agent run. Session tokens enforce per-invocation access boundaries.

Install

From Source

cargo build --release

This produces three binaries:

BinaryRole
opaquedTrusted daemon (enclave, policy, approvals, audit)
opaqueCLI client
opaque-mcpMCP server for Claude Code

Quickstart

Claude Code (MCP)

# 1. Initialize with a preset
opaque init --preset github-secrets

# 2. Start the daemon
opaqued

# 3. Add to Claude Code MCP config
# {
#   "mcpServers": {
#     "opaque": { "command": "/path/to/opaque-mcp" }
#   }
# }

# 4. Ask Claude Code:
#    "Set the GitHub Actions secret API_KEY for myorg/myrepo using my keychain"

CLI / Codex

# Initialize and start
opaque init --preset github-secrets
opaqued

# Test connectivity
opaque ping
opaque execute test.noop

# Sync a GitHub secret
opaque github set-secret \
  --repo myorg/myrepo \
  --secret-name API_KEY \
  --value-ref keychain:opaque/api-key

# Review audit log
opaque audit tail --limit 10

Policy Presets

opaque policy presets                        # list available presets
opaque init --preset safe-demo               # test.noop only
opaque init --preset github-secrets          # GitHub secret sync
opaque init --preset gitlab-variables        # GitLab CI variable sync
opaque init --preset sandbox-human           # sandbox exec for humans only
opaque init --preset agent-wrapper-github    # wrapped-agent GitHub sync
