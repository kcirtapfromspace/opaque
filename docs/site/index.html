How It Works

Every secret operation flows through a five-stage pipeline. The LLM never touches a plaintext value.

The AI agent requests an operation like "set the GitHub secret API_KEY." Opaque checks the policy, prompts you for biometric approval, executes the operation against the provider, strips any secret material from the response, and logs an auditable record. The LLM only ever sees a success/failure confirmation.

Features

Everything you need to let AI tools work with secrets safely.

🔑

Multi-Vault Support

Connect to GitHub, GitLab, 1Password, Bitwarden, HashiCorp Vault, and AWS Secrets Manager. Reference secrets across providers in a single workflow.

🔌

MCP Server

First-class Model Context Protocol integration. Add Opaque to Claude Code's MCP config and your AI assistant can manage secrets through natural language.

🔏

Touch ID Approval

Every secret operation requires explicit biometric approval on macOS. Linux uses polkit for native desktop prompts. No silent access.

📑

Audit Trail

Structured SQLite audit log with correlation IDs, client identity, session tracking, and queryable history. Know exactly what happened and when.

📜

Policy Engine

Deny-by-default with fine-grained allowlist rules. Control which operations each client can perform. Ship with built-in presets for common workflows.

💻

Web Dashboard

Built-in audit analytics dashboard for reviewing operation history, approval patterns, and policy hit rates across all your agent sessions.

Get Started in 60 Seconds

Install Opaque, initialize a preset, start the daemon, and your AI tool is ready to manage secrets safely.

$ brew install anthropics/tap/opaque && opaque init --preset github-secrets && opaqued
